# План разработки

## Цель MVP этапа 1

Собрать рабочий мультиканальный автопостинг для Telegram: источники, генерация поста, дедупликация, базовый фактчек, очередь, ручная модерация, публикация, история и учёт AI usage.

## Вертикальные этапы

1. Каркас проекта: ASP.NET Core backend, PostgreSQL, Redis, React admin, Docker.
2. Домен и БД: каналы, роли, источники, типы публикаций, очередь, история, AI usage, аудит.
3. Источники: Reddit collector с фильтрами subreddit/listing/score/comments/NSFW/keywords.
4. AI-слой: абстракция провайдера, первичная реализация Polza.ai, локальный fallback для dev.
5. Пайплайн поста: кандидат, дедупликация, фактчек, генерация, футер, ручной и автоматический режим публикации.
6. Telegram: публикация в канал и preview на модерацию с кнопками.
7. Админка: dashboard, каналы, источники, очередь, история, расходы.
8. Deployment: docker-compose для API, админки, PostgreSQL и Redis.

## Текущий статус

- Каркас проекта создан.
- Доменная модель MVP создана.
- EF Core контекст и seed первого канала "Только игры" добавлены.
- Reddit collector, базовая дедупликация, базовый фактчек и генератор текста добавлены.
- Polza.ai подключён через provider-абстракцию, dev fallback работает без ключа.
- API endpoints для dashboard, каналов, источников, постов и запуска пайплайна добавлены.
- React-админка для основных MVP-разделов добавлена.
- Docker-контур добавлен.
- Автопилот добавлен: переключатель режима, фоновый worker, лимит постов за прогон, ручной запуск "сейчас" с немедленной публикацией.
